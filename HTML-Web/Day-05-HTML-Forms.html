The Web Needs to Be a Two-Way Street

The fundamental truth is that a website isn't just a brochure for you to read. For the web to be useful, it needs a way to **collect information *from* the user** and send it back to the server. Without this, you couldn't log in, search for a video, buy a product, post a comment, or send a message. The web would be a read-only library.

Without this, you can't log in, search videos, buy stuff, comment, or message anyone. The web would just be a library you can't write in.

The Main Challenge

We need a system to:

  1. Show fields like text boxes or dropdowns for users to fill out.
  2. Pack the user's data neatly.
  3. Send that data to the right place on a server.
  4. Explain to the server how the data is sent.

The answer is the HTML <form> tag.

The Input Element: Your Data Box

The <input> tag is the most common form element. It creates a field for users to enter data, and its behavior depends on the type attribute.

The simplest type is text.

<input type="text">
    
<form>
    <input type="text">
</form>

Adding Meaning with Labels

Problem: A text box alone doesn't tell the user what to enter. Is it for a name, email, or something else?

Solution: Use the <label> tag to describe the field.

Example:

<label>First Name:</label>
<input type="text">

Result: Now users see "First Name:" and know what to type. But the label and input aren't connected yet—the browser doesn't know they belong together.

Connecting Labels and Inputs

Problem: We need to link the label to its input for two reasons:

  1. Usability: Clicking the label should focus the input box.
  2. Accessibility: Screen readers need to know which label goes with which input.

Solution: Use a unique id for the input and match it with the label's for attribute.

  1. Give the input a unique id, like "firstName".
  2. Set the label's for attribute to match that id.

Example:

<label for="firstName">First Name:</label>
<input type="text" id="firstName">

Result:

  • Clicking "First Name:" puts the cursor in the text box.
  • Screen readers say, "First Name, edit text" when focusing the box.

Submitting the Form

Problem: Users can fill out fields, but there's no way to send the data.

Solution:

  1. Wrap all fields in a <form> tag to group them for submission.
  2. Add a submit button with <input type="submit">.

Example:

<form>
    <label for="firstName">First Name:</label>
    <input type="text" id="firstName">
    <br><br>
    <label for="lastName">Last Name:</label>
    <input type="text" id="lastName">
    <br><br>
    <input type="submit">
</form>

Result: You get a form with two fields and a submit button. Clicking the button reloads the page, but the data doesn't go anywhere yet.

Identifying Data with Name

Problem: When the form is submitted, the server needs to know what each piece of data means. If someone types "Arjun" in a box, how does the server know it's the first name? The id is only for the page, not the server.

Solution: Use the name attribute to label data for the server.

Example:

<form>
    <label for="firstName">First Name:</label>
    <input type="text" id="firstName" name="firstName">
    <br><br>
    <label for="lastName">Last Name:</label>
    <input type="text" id="lastName" name="lastName">
    <br><br>
    <input type="submit">
</form>

Result: The form is ready to send data like:

  • firstName = (whatever was typed)
  • lastName = (whatever was typed)

Single Choice with Radio Buttons

Problem: For questions like "What's your T-shirt size (Small, Medium, Large)?", a text box allows inconsistent answers like "Med" or "M".

Solution: Use radio buttons with <input type="radio"> to let users pick one option.

Rule: All radio buttons for one question must share the same name to group them.

Example:

<label>T-Shirt Size:</label>
<br>
<input type="radio" id="sizeS" name="shirtSize" value="small">
<label for="sizeS">Small</label>
<br>
<input type="radio" id="sizeM" name="shirtSize" value="medium">
<label for="sizeM">Medium</label>
<br>
<input type="radio" id="sizeL" name="shirtSize" value="large">
<label for="sizeL">Large</label>
<br><br>

Details:

  • type="radio": Makes a circular button where only one can be selected.
  • name="shirtSize": Groups the buttons so only one can be chosen.
  • id="sizeS": Unique for each label connection.
  • value="small": The data sent to the server if selected, e.g., shirtSize=small.

Multiple Choices with Checkboxes

Problem: For questions like "Which pizza toppings do you want?", users need to pick multiple options.

Solution: Use checkboxes with <input type="checkbox">.

Checkboxes for the same question share the same name to group them for the server.

Example:

<label>Pizza Toppings:</label>
<br>
<input type="checkbox" id="toppingPep" name="toppings" value="pepperoni">
<label for="toppingPep">Pepperoni</label>
<br>
<input type="checkbox" id="toppingMush" name="toppings" value="mushrooms">
<label for="toppingMush">Mushrooms</label>
<br>
<input type="checkbox" id="toppingOni" name="toppings" value="onions">
<label for="toppingOni">Onions</label>
<br><br>

Details:

  • type="checkbox": Creates a square box for multiple selections.
  • name="toppings": Groups all checkboxes under "toppings".
  • id="toppingPep": Unique for label connection.
  • value="pepperoni": The data sent for each selected topping.

Result: Users can select as many toppings as they want.

Better Buttons with Button Tag

Problem: The <input type="submit"> only allows plain text via the value attribute. You can't add images or styled text.

Solution: Use the <button> tag, which can contain other HTML elements.

Example:

<input type="submit" value="Submit Your Order">

<button type="submit">
    <strong>Submit</strong> Your Order
</button>

Details:

  • <button>: A container for the button's content.
  • type="submit": Makes the button submit the form (other types are "button" for JavaScript or "reset").
  • <strong>Submit</strong>: Allows HTML like bold text or images inside.

Result: A more flexible button with the same submit function.

Long Text with Textarea

Problem: The <input type="text"> is bad for long inputs like comments, as text scrolls sideways.

Solution: Use the <textarea> tag for multi-line text.

Example:

<label for="comments">Any special instructions?</label>
<br>
<textarea id="comments" name="comments" rows="4" cols="50"></textarea>
<br><br>
  • rows="4": Sets the height to about 4 lines.
  • cols="50": Sets the width to about 50 characters.

Dropdowns for Many Options

Problem: Radio buttons are fine for a few options, but a long list like countries makes the page too long.

Solution: Use a dropdown with <select> and <option> tags.

Example:

<label for="country">Country:</label>
<br>
<select id="country" name="country">
    <option value="">--Please choose an option--</option>
    <option value="in">India</option>
    <option value="us">USA</option>
    <option value="uk">United Kingdom</option>
    <option value="au">Australia</option>
</select>
<br><br>

Details:

  • <select>: The dropdown container with id and name.
  • <option>: Each choice in the dropdown.
  • value: The data sent to the server; the text (e.g., "India") is what users see.

HTML5 Input Types

HTML5 added new type attributes for smarter forms.

Password Input

Problem: Sensitive data like passwords shouldn't show on screen.

Solution: Use <input type="password"> to mask input.

<label for="userPass">Password:</label>
<br>
<input type="password" id="userPass" name="userPassword">
<br><br>

Number Input with Range

Problem: We want numbers in a specific range, like age, and a number keypad on mobile.

Solution: Use <input type="number"> with min and max.

<label for="userAge">Age (18-99):</label>
<br>
<input type="number" id="userAge" name="age" min="18" max="99">
<br><br>

Result: A number field with up/down arrows, blocking invalid numbers.

Date Input

Problem: Typing dates in a specific format is error-prone.

Solution: Use <input type="date"> for a calendar picker.

<label for="birthDate">Date of Birth:</label>
<br>
<input type="date" id="birthDate" name="dob">
<br><br>

Other Useful Types

  • type="color": Shows a color picker.
  • type="range": Creates a slider.
  • type="file": Lets users upload files.
  • required and placeholder attributes

File Path System

Definition: This section explains how websites use file paths to locate and connect different files like HTML, images, and CSS within a project or across the web.

First Principle

A website is not just one file—it’s a bunch of files (HTML, CSS, images, etc.) organized in folders, like stuff on your computer.

Core Problem

How does index.html find logo.png? You can’t just write src="logo.png" and hope it works. What if it’s in an images folder? The browser needs a clear address—a file path.

Relative File Paths

Definition: Relative file paths describe how to find files starting from your current file’s location, used for linking resources within your website.

Relative paths start from your current file’s location. You’ll use these most of the time for your own files. 


my-website/
├── index.html
├── about.html
├── images/
│   ├── logo.png
│   └── hero.jpg
├── pages/
    ├── contact.html
    └── terms.html

Scenario 1: Same Folder

  • Problem: Link from index.html to about.html.
  • Logic: They’re in the same folder, so just use the filename.
  • Syntax: <a href="about.html">About Us</a>

Scenario 2: Sub-Folder (Down)

  • Problem: Display logo.png from index.html.
  • Logic: Go into the images folder.
  • Syntax: <img src="images/logo.png" alt="Logo">

Scenario 3: Parent Folder (Up)

  • Problem: From contact.html (in pages/), display logo.png (in images/).
  • Logic: Go up from pages/ to my-website/, then down to images/.
  • Syntax: <img src="../images/logo.png" alt="Logo">

Absolute File Paths

Definition: Absolute file paths provide the full URL to access external resources, like images or fonts hosted on other websites.

Absolute paths use the full URL (http:// or https://). Use them for external resources, like Google Fonts or images from another site. 


<a href="https://www.google.com">Google</a>
<img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

Why Avoid Absolute Paths?

Using C:/Users/YourName/... only works on your computer. On a server, it breaks. Browsers also block access to local files for security—no website can sneakily read your files.

Rules of Thumb

  1. Internal files: Use relative paths (filename.html, folder/filename.html, ../filename.html).
  2. External files: Use absolute paths (https://example.com).

Windows Absolute Paths

Windows uses drive letters (C:\) and backslashes (\). Example: C:\Users\JohnDoe\Documents\report.docx.

macOS/Linux Absolute Paths

macOS uses a single root (/) and forward slashes (/). Example: /Users/johndoe/Documents/report.docx.

Key Differences

Feature Windows macOS/Linux
Root C:\ /
Separator \ /
Example C:\Users\JohnDoe\Documents\report.docx /Users/johndoe/Documents/report.docx

HTML Boilerplate

Definition: The HTML boilerplate is a standard template that sets up essential instructions for browsers to render web pages correctly.

First Principle

Browsers need clear instructions to render pages properly—HTML version, language, encoding, etc.

Core Problem

Without setup info, browsers can’t know the HTML version, character encoding, or how to handle mobile displays.

The Boilerplate

A standard template for every HTML file answers these questions. Copy-paste it and tweak as needed.

Boilerplate Breakdown


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
</body>
</html>
  • <!DOCTYPE html>: Tells browser to use HTML5 rules.
  • <html lang="en">: Sets page language for accessibility/SEO.
  • <head>: Holds metadata (not visible on page).
  • <meta charset="UTF-8">: Ensures correct text display.
  • <meta name="viewport">: Makes site mobile-friendly.
  • <title>: Sets browser tab/bookmark name.
  • <body>: Contains visible content.

Conclusion

The boilerplate ensures your site works consistently, is accessible, and mobile-friendly.

Websites Without Boilerplate

Definition: This section explores why browsers can display pages without a boilerplate but why skipping it causes problems.

Forgiving Browsers

Browsers guess to display messy HTML, making the web usable despite errors.

Handling Bad HTML

Browsers assume missing tags, guess encoding, or use filenames as titles, but this can cause issues.

Why Use Boilerplate?

Without it, you risk inconsistent rendering, broken characters, bad mobile display, and poor SEO/accessibility.

Conclusion

Boilerplate is like a solid foundation—skipping it risks a shaky, unreliable website.

Multi-Page Websites

Definition: Multi-page websites organize complex information across multiple pages, connected by navigation, to create a cohesive site.

First Principle

Complex info needs multiple pages, like chapters in a book (e.g., Home, About, Contact).

Core Problem

How to connect pages so they feel like one website, with easy navigation?

Navigation Solution

Use consistent headers/footers and a navigation menu with <a> tags on every page.

<div> (Generic Box)

Definition: The <div> tag groups elements together as a neutral container for styling or layout purposes.

Problem: Need to group elements (e.g., image, heading, paragraph) for styling.

Solution: <div> acts as a neutral container.


<div>
    <img src="avatar.png">
    <h2>Arjun Kumar</h2>
    <p>Loves to code and teach HTML.</p>
</div>

class and id Labels

Definition: Class and id attributes label HTML elements for targeted styling or scripting, with id for unique elements and class for groups.

Problem: How to style specific or groups of <div>s?

Solution: Use id for unique elements, class for groups. 


<div id="main-header">...</div>
<div id="featured-profile" class="profile-card">...</div>
<div class="profile-card">...</div>

<span> (Inline Highlighter)

Definition: The <span> tag highlights or styles small parts of text inline without disrupting the flow of content.

Problem: Style a word in a paragraph without breaking the flow.

Solution: Use <span> for inline styling.


<p>This is very <span class="highlight">important</span> information.</p>

Semantic Tags

Definition: Semantic tags like <header>, <footer>, and <main> add meaning to page structure, improving accessibility and SEO.

  • <header>: Top section (logo, nav).
  • <footer>: Bottom section (copyright, links).
  • <main>: Unique page content (one per page).

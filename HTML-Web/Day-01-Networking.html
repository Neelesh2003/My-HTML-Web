What is Networking and the Internet?

Networking is the connection between computers and devices that allows them to communicate and share information with each other. These connections can be made through cables, wireless signals, or other transmission methods. The Internet is a vast global network made up of millions of smaller networks that are linked together. It enables people from all over the world to access websites, send emails, share files, and use applications in real time. Networking and the Internet rely on protocols like TCP/IP to ensure data is sent and received quickly, securely, and reliably between devices.

Principles of Networking

  • Principle 1 - Sharing Information

    Imagine you have information on one computer and want it on another computer far away. You can use a cable (like ethernet or fiber optic) to send signals. High voltage can mean "1" and low voltage can mean "0". This is how computers communicate in binary.

  • Principle 2 - Scaling Problems

    If you want to connect more than two computers, using cables between every pair is impossible. Instead, we use a Switch that connects all computers in a local area network.

  • Principle 3 - Connecting Networks

    To connect networks across cities or countries, we use Routers. These devices pass data between networks, creating a huge web of connections called the Internet.

  • Principle 4 - Finding Computers

    With millions of computers connected, we need unique addresses. That’s where IP addresses come in. Each device gets a unique address, like a postal address.

  • Principle 5 - Reliability and Fairness

    Large files are broken into packets. Each packet has the sender’s address, receiver’s address, and sequence number. Packets travel independently and are reassembled at the destination. The TCP/IP protocol ensures data arrives correctly.

IP Address (IPv4)

An IP (Internet Protocol) address is a unique numerical label assigned to every device on a network that uses the Internet Protocol, enabling devices to communicate and share data across the internet or local networks. Think of it as a unique postal address for your computer or smartphone, allowing information to be sent to and from the correct location, similar to how mail is delivered to a specific house.

Key Functions:

  • Identification: It uniquely identifies a specific device on a network.
  • Location Addressing: It provides the location information needed to route data to its intended destination

An IPv4 address is a 32-bit number written as four decimal numbers separated by dots.

Example: 172.217.16.142

Total possible addresses: 232 ≈ 4.3 billion.

Public vs Private IP Addresses

Public IP

  • Globally unique address for devices on the Internet.
  • Assigned by Internet Service Providers (ISP).
  • Example: Google's server has a public IP like 142.250.184.142.

Private IP

  • Used inside home or office networks.
  • Can be reused in different networks.
  • Common ranges:
    • 10.0.0.0 - 10.255.255.255
    • 172.16.0.0 - 172.31.255.255
    • 192.168.0.0 - 192.168.255.255
  • Routers use NAT (Network Address Translation) to map private IPs to a public IP.

IPv6

IPv6 solves the problem of running out of addresses. It uses a 128-bit format written in hexadecimal. Example:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Total possible addresses: 2128 ≈ 340 undecillion.

MAC Address

A MAC (Media Access Control) Address is a unique, permanent serial number burned into every network-capable piece of hardware (your laptop's Wi-Fi card, your phone, your smart TV, the network port on your desktop).
To track you, if you get offline and someone get the ip, they will get your message.
A MAC address is a 48-bit number. To make it readable for humans, it's typically written as **12 hexadecimal digits**.

A MAC (Media Access Control) address is a unique hardware identifier assigned to network devices.

Example formats:

  • 3C:22:FB:A3:B4:C5 (colon-separated)
  • 3C-22-FB-A3-B4-C5 (hyphen-separated)
  • 3c22.fba3.b4c5 (period-separated)

Finding MAC Address

  • Windows: Run ipconfig /all or getmac /v.
  • macOS/Linux: Run ifconfig or ip link.

Port Numbers (0–65535)

Port numbers are numerical identifiers (0-65535) in computer networking that identify specific applications or services on a device. They work with IP addresses to direct network traffic to the correct service on a machine, with common ports assigned to standard applications like HTTP (port 80) and HTTPS (port 443).

Ports allow multiple applications to use the same IP address. Each service uses a specific port number.

A port number is a 16-bit unsigned integer.

Let's break that down:
  • 16-bit:This means the number is stored using 16 binary digits (1s and 0s).
  • Unsigned:This means it cannot be negative. The values are all positive.
Because it's a 16-bit number, the total number of possible ports is 2^16, which equals **65,536**.

Categories

  • Well-known ports (0-1023): Reserved for standard services.
  • Registered ports (1024-49151): For user applications.
  • Dynamic ports (49152-65535): Temporary ports assigned by the OS.

Examples

Service Port
HTTP 80
HTTPS 443
SSH 22
MySQL 3306
MongoDB 27017

DNS — Domain Name System

DNS converts names like www.google.com into IP addresses like 142.250.72.206.

DNS, or the Domain Name System, acts as the internet's phone book, translating human-readable domain names (like www.google.com) into numerical IP addresses (like 172.217.160.142) that computers use to locate websites and other online resources. Without DNS, you would have to memorize complex IP addresses for every website you wanted to visit, but DNS simplifies this by providing a human-friendly name-to-number mapping system, allowing you to just type a website's name and be connected to the correct server.

How DNS Works

  1. Check browser cache, OS cache, or router cache first.
  2. If not found, ask recursive resolver (ISP or public DNS like 8.8.8.8).
  3. The resolver asks root servers → TLD servers → authoritative servers.
  4. The answer is returned and cached for future requests.

Types of DNS Servers

  • Root Servers: Know where to find top-level domains like .com or .org.
  • TLD Servers: Manage domain extensions.
  • Authoritative Servers: Hold records for specific websites.
  • Recursive Resolvers: Ask questions on your behalf and find answers.

Common DNS Records

Record Type Description
A Maps name to IPv4 address
AAAA Maps name to IPv6 address
CNAME Alias to another domain name
MX Email server for the domain
NS Name server for the domain
TXT Verification or security information

History of the Internet

  • 1969: ARPANET connects 4 universities.
  • 1983: TCP/IP becomes the standard protocol.
  • 1990: Tim Berners-Lee creates the World Wide Web.
  • 1991: First website goes live at info.cern.ch.
  • Today: Billions of devices connect using Wi-Fi, fiber optics, satellites, and mobile networks.

Web Technologies — HTML, CSS, JavaScript

HTML
Structures content using headings, paragraphs, links, and tables. It defines the layout and elements on a webpage.
CSS
Adds styling like fonts, colors, layouts, and spacing. CSS ensures that websites look attractive and responsive across different devices.
JavaScript
Makes websites interactive with animations, buttons, forms, and dynamic content. It allows websites to respond to user actions in real-time.

How Web Technologies Work with Networking

Web pages are stored on servers connected through the Internet. When a user visits a website:

  • The browser sends a request to the server using networking protocols like HTTP or HTTPS.
  • The server responds with the website’s HTML, CSS, and JavaScript files.
  • The browser renders the content and applies styling and interactivity based on these files.
  • JavaScript can make further network requests using APIs like fetch() or XMLHttpRequest to update content without reloading the page.

Popular JavaScript Frameworks

Frameworks help developers build complex websites and applications faster, while managing code efficiently and improving user experience.

React

  • A library for building user interfaces using reusable components.
  • Uses a virtual DOM to update only parts of the page that change, making applications faster.
  • Works seamlessly with APIs to fetch or send data over the network.

Vue

  • Progressive framework for building interactive interfaces.
  • Uses two-way data binding to reflect changes instantly on the page.
  • Ideal for small to medium projects and integrates easily with existing applications.

Angular

  • A powerful framework that supports large, enterprise-level applications.
  • Offers tools for routing, state management, and form handling.
  • Uses dependency injection and modular architecture to improve scalability.
  • Communicates with servers via HTTP services to fetch or store data.

Web Technology + Networking Example

Imagine an online shopping site:

  1. The user’s browser requests the product page from the server.
  2. The server sends back the HTML structure and CSS styles.
  3. JavaScript loads product details from a database by making network requests.
  4. React or Angular dynamically updates the page when users add items to the cart or filter products.
  5. All interactions happen smoothly without reloading the entire page.

Summary

  • The Internet is a network of networks connected by routers.
  • IP addresses uniquely identify devices; IPv6 solves address exhaustion.
  • Data is broken into packets using protocols like TCP/IP.
  • MAC addresses identify devices locally, while IP addresses are used globally.
  • Port numbers help route traffic to specific applications.
  • DNS helps translate domain names to IP addresses quickly and efficiently.
